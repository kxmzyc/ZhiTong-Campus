1. 通用说明
1.1 统一响应格式
所有接口均返回统一的 JSON 结构：

JSON

{
  "code": 200,      // 200 表示成功，其他状态码表示异常
  "msg": "操作成功", // 提示信息
  "data": { ... }   // 业务数据
}
1.2 异常状态码
200: 成功

400: 参数错误

401: 未登录/Token过期

403: 无权限

500: 服务器内部错误

3. 用户模块 (User)
3.1 用户登录 (微信小程序)
接口地址: /user/login

请求方式: POST

参数:

JSON

{
  "code": "081......", // wx.login 获取的临时凭证
  "userInfo": { "nickName": "...", "avatarUrl": "..." } // 可选，用于自动注册
}
3.2 获取用户信息
接口地址: /user/{id}

请求方式: GET

3.3 更新用户信息
接口地址: /user

请求方式: PUT

参数:

JSON

{
  "id": 1,
  "nickname": "林同学",
  "mobile": "13800138000",
  "email": "lin@example.com"
}
3.4 获取积分余额
接口地址: /user/points/{userId}

请求方式: GET

响应: { "balance": 120 }

3.5 积分充值/扣减
接口地址: /user/points/add 或 /user/points/deduct

请求方式: POST

参数: { "userId": 1, "points": 10 }

4. 简历模块 (Resume)
核心业务模块，包含简历主体及子模块。

4.1 获取简历列表
接口地址: /resume/list/{userId}

请求方式: GET

4.2 获取简历详情 (基本信息)
接口地址: /resume/{id}

请求方式: GET

4.3 获取简历完整预览 (聚合接口)
重要：此接口用于“简历预览”页，需一次性返回简历主体及所有关联表数据。

接口地址: /resume/preview/{id}

请求方式: GET

响应结构:

JSON

{
  "code": 200,
  "data": {
    "resume": { "id": 1, "name": "...", "jobIntent": "Java开发" },
    "educationList": [ { "school": "...", "major": "..." } ],
    "projectList": [ { "projectName": "...", "role": "..." } ],
    "internshipList": [],
    "awardList": []
  }
}
4.4 创建/更新简历 (基本信息)
创建: POST /resume

更新: PUT /resume

参数: 参见 Resume 实体类

4.5 删除简历
接口地址: /resume/{id}

请求方式: DELETE

4.6 简历子模块管理
包含：教育经历 (education)、项目经历 (project)、实习经历 (internship)、获奖经历 (award)。

以教育经历为例 (其他模块路径类似，只需替换 /education 为对应模块名)：

获取列表: GET /resume/education/list/{resumeId}

新增经历: POST /resume/education

更新经历: PUT /resume/education

删除经历: DELETE /resume/education/{id}

4.7 AI 简历诊断 [待实现]
接口地址: /resume/analyze

请求方式: POST

参数: { "resumeId": 1 }

响应: 返回评分及建议文本。

5. 职位模块 (Job)
5.1 职位列表 (分页)
接口地址: /job/list

请求方式: GET

参数: page, size

接口地址: /job/{id}

5.4 职位高级搜索 [待实现]
接口地址: /job/search

请求方式: GET

参数: keyword, city, salaryRange, minDegree

6. 交互模块 (Interaction)
6.1 投递职位
接口地址: /application/apply

请求方式: POST

参数: { "userId": 1, "jobId": 101, "resumeId": 50 }

6.2 投递记录查询
接口地址: /application/list/{userId}

请求方式: GET

参数: status (可选：0-已投递, 1-被查看, 2-面试, 3-不合适)

6.3 职位收藏
添加收藏: POST /favorite/add

取消收藏: DELETE /favorite/remove

状态检查: GET /favorite/check (查询用户是否收藏了某职位)

收藏列表: GET /favorite/list/{userId}
