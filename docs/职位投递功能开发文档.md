#### 1.2 修改 `onLoad` 方法

在页面加载时，除了加载职位详情，还会检查投递状态：

```javascript
onLoad(options) {
  const jobId = options.id ? parseInt(options.id) : null;
  if (jobId) {
    this.setData({ jobId });
    this.loadJobDetail(jobId);
    // 检查是否已投递
    this.checkApplicationStatus(jobId);
  }
}
```

#### 1.3 新增 `checkApplicationStatus` 方法

检查用户是否已投递该职位：

```javascript
checkApplicationStatus(jobId) {
  const userId = this.data.userId;

  api.checkApplied(userId, jobId)
    .then(res => {
      if (res.code === 200) {
        const isApplied = res.data === true || res.data === 'true';

        this.setData({
          isApplied: isApplied,
          applyBtnText: isApplied ? '已投递' : '投递简历',
          applyBtnDisabled: isApplied
        });
      }
    })
    .catch(err => {
      console.error('❌ 检查投递状态失败:', err);
      // 检查失败不影响页面展示，默认为未投递
    });
}
```

#### 1.4 重写 `onApply` 方法

实现完整的投递逻辑：

```javascript
onApply() {
  // 1. 防止重复投递
  if (this.data.isApplied) {
    wx.showToast({
      title: '您已投递过该职位',
      icon: 'none'
    });
    return;
  }

  // 2. 获取参数
  const userId = this.data.userId;
  const jobId = this.data.jobId;
  const resumeId = this.data.resumeId;

  // 3. 显示加载状态
  wx.showLoading({ title: '投递中...' });

  // 4. 调用投递接口
  api.applyJob(userId, jobId, resumeId)
    .then(res => {
      wx.hideLoading();
      if (res.code === 200) {
        // 5. 更新投递状态
        this.setData({
          isApplied: true,
          applyBtnText: '已投递',
          applyBtnDisabled: true
        });

        // 6. 提示成功
        wx.showToast({
          title: '投递成功',
          icon: 'success',
          duration: 2000
        });
      } else {
        throw new Error(res.message || '投递失败');
      }
    })
    .catch(err => {
      wx.hideLoading();
      // 7. 错误处理
      const errorMsg = err.message || err.errMsg || '投递失败，请稍后重试';
      wx.showToast({
        title: errorMsg,
        icon: 'none',
        duration: 2000
      });
    });
}
```

### 2. 修改 `pages/job-detail/job-detail.wxml`

更新投递按钮，支持动态文字和禁用状态：

```xml
<!-- 修改前 -->
<button class="btn-apply" bindtap="onApply">投递简历</button>

<!-- 修改后 -->
<button
  class="btn-apply {{isApplied ? 'btn-applied' : ''}}"
  bindtap="onApply"
  disabled="{{applyBtnDisabled}}">

</button>
```

**关键属性：**
- `class`: 根据 `isApplied` 动态添加 `btn-applied` 样式
- `disabled`: 绑定 `applyBtnDisabled` 控制按钮禁用
- 按钮文字：绑定 `applyBtnText`，动态显示"投递简历"或"已投递"

## 🔗 API 接口

### 1. 检查投递状态

```
GET /api/application/check?userId={userId}&jobId={jobId}
```

**请求参数：**
- `userId`: 用户ID（当前写死为 1）
- `jobId`: 职位ID

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": true  // true-已投递，false-未投递
}
```

### 2. 投递简历

```
POST /api/application/apply
```

**请求体：**
```json
{
  "userId": 1,
  "jobId": 1,
  "resumeId": 1
}
```

**响应示例：**
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": 1,
    "userId": 1,
    "jobId": 1,
    "resumeId": 1,
    "status": "pending"
  }
}
```

## 📊 状态流转

### 按钮状态变化

```
初始状态
  ↓
检查投递状态 (checkApplicationStatus)
  ↓
├─ 未投递 → 按钮：蓝色"投递简历"（可点击）
└─ 已投递 → 按钮：灰色"已投递"（禁用）
  ↓
用户点击投递
  ↓
调用投递接口 (applyJob)
  ↓
├─ 成功 → 按钮变为：灰色"已投递"（禁用）
└─ 失败 → 提示错误，按钮保持可点击
```

|------|-----------|--------------|------------------|----------|
| 未投递 | false | "投递简历" | false | 蓝色渐变 |
| 已投递 | true | "已投递" | true | 灰色 |

## ⚠️ 注意事项

### 1. 测试数据

当前使用写死的测试数据：
- `userId: 1` - 测试用户
- `resumeId: 1` - 测试简历

**生产环境需要修改为：**
```javascript
// 从全局状态或缓存中获取真实用户ID
const userId = wx.getStorageSync('userId') || getApp().globalData.userId;

// 让用户选择简历
const resumeId = this.selectResume();
```

### 2. 投递状态检查

- 页面加载时自动检查投递状态
- 检查失败不影响页面展示，默认为未投递
- 避免因网络问题导致用户无法投递

### 3. 防重复投递

三重保护：
1. **前端状态检查**：`if (this.data.isApplied) return;`
2. **按钮禁用**：`disabled="{{applyBtnDisabled}}"`
3. **后端验证**：后端应该也要检查是否重复投递

### 4. 错误处理

- 投递失败时显示具体错误信息
- 网络错误时提示"投递失败，请稍后重试"
- 按钮保持可点击状态，允许用户重试
