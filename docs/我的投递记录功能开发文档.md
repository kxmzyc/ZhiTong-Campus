### 一、后端开发 (Java Spring Boot)

#### 1. 创建 VO 类

**文件：`vo/JobApplicationVO.java`**

包含字段：
- `id` - 申请ID
- `jobId` - 职位ID
- `jobTitle` - 职位名称
- `salaryMin` / `salaryMax` - 薪资范围
- `companyName` - 公司名称

- `status` - 申请状态
- `appliedAt` - 申请时间
- `city` - 工作城市
- `companyId` - 公司ID

#### 2. 创建 Mapper XML

**文件：`resources/mapper/JobApplicationMapper.xml`**

实现多表关联查询：
```sql
SELECT
    ja.id,
    ja.job_id,
    j.title AS job_title,
    j.salary_min,
    j.salary_max,
    j.city,
    c.id AS company_id,
    c.name AS company_name,
    c.logo AS company_logo,
    ja.status,
    ja.applied_at
FROM
    job_application ja
LEFT JOIN
    job j ON ja.job_id = j.id
LEFT JOIN
    company c ON j.company_id = c.id
WHERE
    ja.user_id = #{userId}
    AND ja.deleted = 0
ORDER BY
    ja.applied_at DESC
```

#### 5. 修改 Controller

**文件：`controller/JobApplicationController.java`**

新增接口：
```java

```

**接口路径：** `GET /api/application/list?userId={userId}`

## 📊 数据流程

### 1. 页面加载流程

```
用户进入投递记录页
    ↓
调用 loadApplications()
    ↓
请求 GET /api/application/list?userId=1
    ↓
```

## ⚠️ 注意事项

### 1. 测试数据

当前使用写死的用户ID：
```javascript
userId: 1
```

**生产环境需要修改为：**
```javascript
const userId = wx.getStorageSync('userId') || getApp().globalData.userId;
```

### 2. 多表关联查询

- 使用 LEFT JOIN 确保即使公司或职位被删除，投递记录仍然可见
- 添加 `deleted = 0` 条件过滤逻辑删除的记录
- 按投递时间倒序排列

### 3. 状态管理

- 只有 `pending` 状态的申请可以撤销
- 其他状态（已查看、面试中等）不显示撤销按钮

### 4. 错误处理

- 接口失败时显示错误提示
- 加载状态提示用户等待
- 空状态友好提示
