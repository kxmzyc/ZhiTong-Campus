# IDEA 数据源配置指南

## 方法一：在 IDEA 中配置数据源（推荐）

### 步骤1：打开数据库工具窗口
1. 在 IDEA 右侧边栏找到 **Database** 标签
2. 如果没有看到，点击菜单 **View → Tool Windows → Database**

### 步骤2：添加数据源
1. 点击 Database 窗口左上角的 **+** 号
2. 选择 **Data Source → MySQL**

### 步骤3：配置连接信息
在弹出的配置窗口中填写：

```
Name: zhitongxiaoyuan
Host: localhost
Port: 3306
Database: zhitongxiaoyuan
User: root
Password: 041029
```

### 步骤4：下载驱动（如果需要）
1. 如果提示缺少驱动，点击 **Download missing driver files**
2. 等待驱动下载完成

### 步骤5：测试连接
1. 点击 **Test Connection** 按钮
2. 如果显示 **Successful**，说明连接成功
3. 点击 **OK** 保存配置

### 步骤6：运行SQL脚本
1. 在 IDEA 中打开文件 `快速测试数据.sql`
2. 右键点击编辑器
3. 选择 **Execute** 或按快捷键 **Ctrl+Enter**
4. 选择刚才配置的数据源 **zhitongxiaoyuan**
5. 点击 **Execute** 执行

## 方法二：使用命令行直接执行

### 步骤1：找到 MySQL 安装目录
通常在以下位置之一：
- `C:\Program Files\MySQL\MySQL Server 8.0\bin`
- `C:\Program Files (x86)\MySQL\MySQL Server 8.0\bin`
- `C:\xampp\mysql\bin`
- `C:\wamp\bin\mysql\mysql8.0.x\bin`

### 步骤2：打开命令提示符
1. 按 **Win + R**
2. 输入 `cmd`
3. 按回车

### 步骤3：进入 MySQL bin 目录
```cmd
cd "C:\Program Files\MySQL\MySQL Server 8.0\bin"
```
（根据实际安装路径调整）

### 步骤4：登录 MySQL
```cmd
mysql -uroot -p041029
```

### 步骤5：执行 SQL 脚本
```sql
-- 切换到数据库
use zhitongxiaoyuan;

-- 执行脚本（方式1：直接在命令行中）
source G:/claude/ZhiTongXiaoYuan/快速测试数据.sql;

-- 或者（方式2：手动复制粘贴SQL内容）
-- 打开 快速测试数据.sql 文件，复制所有内容，粘贴到命令行中执行
```

## 方法三：使用 Navicat 或其他数据库工具

### 如果你有 Navicat
1. 打开 Navicat
2. 点击 **连接 → MySQL**
3. 填写连接信息：
   - 连接名：zhitongxiaoyuan
   - 主机：localhost
   - 端口：3306
   - 用户名：root
   - 密码：041029
4. 点击 **测试连接**
5. 连接成功后，双击打开连接
6. 选择数据库 **zhitongxiaoyuan**
7. 点击 **查询 → 新建查询**
8. 打开 `快速测试数据.sql` 文件，复制内容
9. 粘贴到查询窗口
10. 点击 **运行** 或按 **F5**

### 如果你有 MySQL Workbench
1. 打开 MySQL Workbench
2. 点击已有的连接或创建新连接
3. 填写连接信息（同上）
4. 连接成功后，点击 **File → Open SQL Script**
5. 选择 `快速测试数据.sql` 文件
6. 点击工具栏的 **闪电图标** 执行脚本

## 常见问题

### 1. 找不到 Database 窗口
- 点击 IDEA 菜单 **View → Tool Windows → Database**
- 或者按快捷键 **Alt+1** 然后选择 Database

### 2. 连接测试失败
- 检查 MySQL 服务是否启动
- 检查用户名密码是否正确（root / 041029）
- 检查端口是否为 3306

### 3. 驱动下载失败
- 检查网络连接
- 或者手动下载 MySQL Connector/J 驱动
- 放到 IDEA 的 lib 目录中

### 4. 执行 SQL 报错
- 确保已经选择了正确的数据库
- 确保数据库表已经创建（运行过 schema.sql）
- 检查是否有重复数据（如果已经插入过，删除后重新插入）
