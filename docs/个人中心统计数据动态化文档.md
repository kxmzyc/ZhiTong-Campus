# 个人中心统计数据动态化开发文档

## 📋 功能概述

将小程序"我的"页面的写死数据（投递过、待网申、已收藏）改为从后端动态获取，实现实时统计展示。

## ✅ 已完成的工作

### 一、后端开发 (Java Spring Boot)

#### 1. 修改 UserController

**文件：`controller/UserController.java`**

**新增依赖注入：**
```java
@Autowired
private JobApplicationService jobApplicationService;

@Autowired
private FavoriteService favoriteService;
```

**新增统计接口：**
```java
@GetMapping("/stats")
public Result<Map<String, Integer>> getUserStats(@RequestParam Long userId) {
    Map<String, Integer> stats = new HashMap<>();

    // 统计投递过的数量（所有状态）
    int appliedCount = jobApplicationService.getApplicationsByUserId(userId).size();

    // 统计待处理的数量（pending 状态）
    int pendingCount = jobApplicationService.getApplicationsByUserIdAndStatus(userId, "pending").size();

    // 统计收藏的数量
    int favoriteCount = favoriteService.getFavoritesByUserId(userId).size();

    stats.put("appliedCount", appliedCount);
    stats.put("pendingCount", pendingCount);
    stats.put("favoriteCount", favoriteCount);

    return Result.success(stats);
}
```

**接口路径：** `GET /api/user/stats?userId={userId}`

```javascript
checkLoginStatus() {
  const isLogin = app.checkLogin();
  this.setData({ isLogin });

  if (isLogin) {
    // 已登录，加载用户信息和统计数据
    this.loadUserInfo();
    this.loadUserStats();
  } else {
    // 未登录，显示默认信息
    this.setData({
      // ...
      stats: {
        applied: 0,
        pending: 0,
        favorite: 0
      }
    });
  }
}
```

**新增 loadUserStats 方法：**
```javascript
/**
 * 加载用户统计数据
 */
loadUserStats() {
  // 获取用户ID
  const userId = app.globalData.userId || wx.getStorageSync('userId') || 1;

  api.getUserStats(userId)
    .then(res => {
      if (res.code === 200 && res.data) {
        this.setData({
          stats: {
            applied: res.data.appliedCount || 0,
            pending: res.data.pendingCount || 0,
            favorite: res.data.favoriteCount || 0
          }
        });
      }
    })
    .catch(err => {
      console.error('加载统计数据失败:', err);
      // 失败时保持默认值 0
    });
}
```

## 📊 数据流程

### 页面加载流程

```
用户进入"我的"页面
    ↓
onShow() 触发
    ↓
checkLoginStatus()
    ↓
检查是否登录
    ↓
├─ 未登录 → 显示默认值 0
└─ 已登录 → 调用 loadUserStats()
    ↓
请求 GET /api/user/stats?userId={userId}
    ↓
后端统计数据
    ↓
返回统计结果
    ↓
前端更新显示
```

### 统计逻辑

| 统计项 | 后端实现 | 说明 |
|--------|---------|------|
| 投递过 (appliedCount) | `getApplicationsByUserId()` | 所有状态的投递记录 |
| 待网申 (pendingCount) | `getApplicationsByUserIdAndStatus("pending")` | 仅 pending 状态 |
| 已收藏 (favoriteCount) | `getFavoritesByUserId()` | 所有收藏记录 |

## ⚠️ 注意事项

### 1. 用户ID获取

当前使用多种方式获取用户ID：
```javascript
const userId = app.globalData.userId || wx.getStorageSync('userId') || 1;
```

**优先级：**
1. `app.globalData.userId` - 全局状态
2. `wx.getStorageSync('userId')` - 本地缓存
3. `1` - 默认测试ID

### 2. 未登录处理

- 未登录时显示默认值 0
- 不会调用后端接口
- 避免不必要的请求

### 3. 错误处理

- 接口失败时保持默认值 0
- 不影响页面其他功能
- 控制台输出错误日志

### 4. 性能优化

当前实现使用 `size()` 统计数量，对于大数据量可能影响性能。

**优化建议：**
```java
// 使用 count 查询代替 list().size()
int appliedCount = (int) jobApplicationService.count(
    new QueryWrapper<JobApplication>().eq("user_id", userId)
);
```
